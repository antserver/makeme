Building the MPR

Here is a portion of an MPR bit file that two libraries and one executable. It uses the ${BIN} global token which expands into the platform build directory. The ${EXE} token will expand to ".exe" on windows. The gui rule builds a Windows application instead of a console application on Windows.

Bit.load({
    targets: {
        libmpr: {
            type: 'lib',
            sources: [ 'mprLib.c' ],
            headers: [ '*.h' ],
        },      
        manager: {
            enable: 'bit.settings.mpr && bit.settings.mpr.manager',
            type: 'exe',
            path: '${BIN}/${settings.mpr.manager}${EXE}'
            rule: 'gui',
            depends: [ 'libmpr' ],
            sources: [ 'manager.c' ],
        },
    },
})

Building Ejscript

Here is a bit file from Ejscript that adds a library dependency at runtime.

Bit.load({
    targets: {
        ejs: {
            type: 'exe',
            sources: [ 'ejs.c', ],
            depends: [ 'libejs' ],
            scripts: {
                prebuild: "
                    /* If the system has libedit, add it here */
                    if (bit.settings.hasLibEdit) {
                        bit.target.libraries.push('edit')
                    }
                ",
            },
        },
    },
})
