Scripts

Bit includes a powerful Javascript engine based on the Ejscript language environment. This allows fast, and powerful scripting. Ejscript has an extensive Library of routines that are ideal for building projects.

Bit target scripts may be specified three ways:

  • Shell scripts — bash for Unix systems
  • Build target scripts — Ejscript for portable scripts
  • Action target scripts — Ejscript for portable scripts
  • Long-hand scripts — Custom script interpreters

Quoting

First a general note about quoting. Scripts can be expressed on one line or over multiple lines. You can choose whether you use single quotes or double quotes around your script. Inside the script you can then freely use the alternate quote character without needing to back-quote. If you need to use the outside quote within the script, quote it with a backslash \ character.

It is typically best to use single quotes around a shell script so that double quotes can be used inside. For example:

shell: 'echo "Dont\'t panic"',

Shell Scripts

Shell scripts can be specified via the target shell property: 

cleanup: {
    shell: '
        date >now.txt 
        echo hello
        echo world
    ',
},

This script will be run when the named target is invoked.

bit cleanup

If you need a shell script to run when a default build is initiated, see the Long-hand Scripts section below.

Action Scripts

Action scripts use the Ejscript (Javascript) language for portable execution on any platform. Bit has an internal Ejscript interpreter and so it can execute actions very quickly without needing to spawn another process.

Action scripts are defined using the action property. For example:

targets: {
    'generate-doc': {
        build: "
            Path('build-state').write('Started at ' + Date())
        ",
    }
}
Like shell script targets, actions do not run by for default builds. Rather, they are run when they are explicitly invoked on the command line or another target is invoked that references them as a dependency.

Build Scripts

Build scripts are a convenient way to express actions that should run for default builds. Build scripts use Ejscript and are defined via the build property. For example:

'building': {
    build: "
        Http().post('http://example.com/buildbot', 'building')
    ",
}

When bit is invoked, the script for the building target will be run. When the target runs will be dictated by whether other targets depend upon it. Note that in Ejscript, quoted literals can span multiple lines. No backquoting at the end of the line is required. Also, Ejscript permits string literals to span multiple lines.

Target Paths

You can make a build script only run when it is out-of-date by adding a path property for a physical file that will be created by the target. In this way, the target will only be run if the designated file does not exist or another target that depends on this target is more up-to-date than the file.

'build-complete': {
    path: 'complete.tmp',
    build: "
        Path('complete.tmp').write('Completed at: ' + Date())
    ",
}

When bit generate-doc is run, this generate-documentation script will be run.

Long-hand Scripts

The shell, action and build scripts are actually abbreviated ways to express a more generalized, and more powerful underlying target scripts capability. Each target can define scripts that run at various stages of the build lifecycle. The shell, action, and build properties are built atop this facility.

The scripts collection is an object hash with properties for life cycle events. When the specific event is triggered, the script is run. For each event, there can be multiple scripts which are run in-order. Each script has a home directory which becomes the current directory for the script. It also specifies the interpreter to execute the script. 

'build-complete': {
    path: 'complete.tmp',
    type: 'action',
    scripts: {
        build: [ 
            {
                home: '.',
                shell: 'ejs',
                script: "
                    Path('complete.tmp').write('Completed at: ' + Date())
                ",
            },
        ],
    },
},

Note that the build and action scripts translate under-the-hood into this long form. An action script becomes a target with a type of action and an event of build. A build script becomes a target with a type of build and an event of build.

The shell property specify the name of the interpreter to use and will typically be set to ejs for the internal Ejscript interpreter or bash for an external Unix shell interpreter. The home property specifies the home directory in which to run the script which defaults to the directory in which the bit file resides.

The target scripts property is a collection indexed by a by when the script should run. This will be either:

WhenDescription
preblendRun before blending defaults and internal properties into the target
preresolveRun before resolving the target's dependent libraries
postresolveRun after resolving the target's dependent libraries
presourceRun before expanding the sources property wildcards
prebuildRun just before building the target
postblendRun after blending defaults into the target properties
buildRun when the target should be built
postconfigRun after configuration but before generating the bit.h header
predependenciesRun before the dependencies for a target are built
precompileRun before a source file is compiled

Note that multiple scripts can be specified for each lifecycle phase.

Here is another example of a shell script that runs just prior to building a library. 

libglow: {
    type: 'lib',
    sources: '*.c',
    scripts: {
        prebuild: [ {
            shell: 'bash',
            script: './build-prep',
        ],
    }
}

Useful Scripting Bits

Here are some useful bits of Ejscript to use in your scripts:

List of Files

Get a list of all files in all subdirectories 

let files = Path('some/dir').files('**')

Run an external command

Get a list of all files in all subdirectories 

run('/usr/bin/program')

Write a file

Get a list of all files in all subdirectories 

Path('filename').write('Hello World')

Read a file

Read a file into a variable as a string 

let data = Path('filename').readString

Read a JSON file

Read a JSON file as an object 

let name = Path('filename').readJSON().name

Work with Paths

Here are some useful Path manipulation routines. Ejscript is great at this. 

let p = Path('/some/file.c')
print('Basename', p.basename)
print('Dirname', p.dirname)
print('Extension', p.extension)
print('Joining', p.dirname.join('other.c'))

See the Ejscript Path class for full details. Similarly, for URLs, see: URI class, and for Http requests, see the Http class.
