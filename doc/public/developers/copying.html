Copying Files
v0.8.3

Copy Files

You can copy files at build time by using a copy target which specifies a set of input files and an output destination. The copy target has a type property set to 'file'. For example:

setup: {
    type: 'file',
    path: '${CFG}/web',
    files: [ 'web/readme.html' ],
},

This example will copy the specified readme file to the web directory under the platform output directory.

If the destination path property is a directory, the specified files will be copied into that directory. If the destination path has a trailing "/", it will be assumed to be a directory and will be created if it does not exist.

The files property is an array of files or directories. These files can include wild cards which are expanded using the Ejscript Path.files routine. The supported wild cards are:

  • ? — matches any single character
  • * — matches zero or more characters in a filename or directory
  • ** — zero or more files or directories and matches recursively in a directory tree
  • / — If a pattern terminates with / it will only match directories.

If an item in the files property is a directory, then the directory and all the files in that directory will be copied. In other words, the item will automatically be converted to path/**. If the item has a trailing "/" then the contents of the directory will be copied and not the directory itself.

If the destination path if up-to-date, meaning that no other target that depends on this target is more current, then the files will not be copied.

To learn more, read about Creating a Configurable Project.
