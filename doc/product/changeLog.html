Bit Change Log

Brief Bit change log of the the major features and changes for each release.

Note to security researchers:

Please do not mine this list and create security advisories. Please see the Bit Issue Database for the official product issue list and for comprehensive CVE dispositions.

Changes in 0.8.4

Minor Features

  • Add target.run property to run commands [issue 8].
  • Allow array of target.require and manifest.file.require [issue 9].
  • Allow array of bit.scripts [issue 10].
  • Add --configuration switch and configuration target to display current configuration [issue 10].

Fixes

  • Rename platform.os == 'posix' to 'unix' [issue 11].
  • Refactor event names [issue 12].

Changes in 0.8.3

Versions 0.8.1 and 0.8.2 were internal releases.

Minor Features

  • More documentation [issue 7].

Fixes

  • Fix make uninstall [issue 6].

Changes in 0.8.3

Major Features

  • First stand-alone release for Bit. Bit separated from Ejscript as a stand-alone product. [issue 2].

Minor Features

  • Enhanced Makefile generation [issue 1].

Fixes

  • Fix generated makefiles determining the system CPU architecture [issue 3].
  • Remove compiler switch -mtune when not required [issue 4].
  • Bit support for static projects via configure --static, including generating IDE projects [issue 5].
